# Nix flake dev shell

## Objective

Author `flake.nix` (and check in `flake.lock`) that provides a reproducible dev shell containing every host-level binary symphony depends on: `bun`, `nono`, `claude`, plus standard build tools. Running `nix develop` (or using `direnv` with `.envrc` → `use flake`) drops you into a shell where every following task's commands just work.

## Context

### Parent PRD

- **PRD**: Symphony v1 (Effect.ts + Claude Code)
- **PRD Path**: `.claude/prds/symphony-v1/PRD.md`

### Related Tasks

- **Blocks**: every other task. Nothing else runs reliably outside the flake.

## Acceptance Criteria

- [ ] `flake.nix` exists at repo root with `inputs.nixpkgs` pinned to an explicit revision (not `nixos-unstable`'s rolling alias) and `inputs.flake-utils` for multi-system support.
- [ ] `devShells.${system}.default` provides at minimum: `bun`, `nono`, the `claude` CLI (>= 2.0.0 per the stream-json minimum, see `research/claude-stream-json.md`), `git`, `jq`, `yq`.
- [ ] `flake.lock` is committed.
- [ ] `nix develop --command bun --version` and `nix develop --command nono --version` and `nix develop --command claude --version` all exit 0.
- [ ] Works on `aarch64-darwin` (Seth's laptop) and `x86_64-linux` (eventual remote box).
- [ ] If `claude` is not yet packaged in nixpkgs, vendor it as a derivation in the flake (downloading the binary from Anthropic's release channel) rather than asking users to install it out-of-band.
- [ ] A short `flake.nix`-level comment explains the build inputs at a glance.

## Implementation Notes

### Files to Modify

| File Path | Action | Description |
|-----------|--------|-------------|
| `flake.nix` | Create | Dev shell definition. |
| `flake.lock` | Create | Generated by `nix flake update`. |
| `.envrc` | Create | `use flake` (optional but convenient for direnv users). |

### Technical Constraints

- Pin nixpkgs to a specific commit. Do not float.
- Don't use `mkShell` with overrides that mask `bun` from PATH.
- `pkgs.nono` is available in nixpkgs (per `research/nono-sh.md`). If the available version is older than what we need, override via `overlay`.
- The `claude` CLI may need a custom derivation. If so, use `stdenv.mkDerivation` with `srcs = fetchurl { url = ...; sha256 = ...; }`. Mark the package `meta.platforms = [ "aarch64-darwin" "x86_64-linux" "aarch64-linux" ]`.

### Relevant Code References

- `research/nono-sh.md` — confirms `pkgs.nono` exists in nixpkgs.
- `research/claude-stream-json.md` §1 — minimum CLI version 2.0.0 (`subprocess_cli.py:31`).

## Testing Requirements

- [ ] `nix flake check` passes.
- [ ] CI (if added) installs Nix and runs `nix develop --command bun test` for the rest of the suite.

## Out of Scope

- A production-grade NixOS module or systemd unit for deploying symphony to a remote host (the PRD defers remote deployment to post-v1).
- Cross-compilation. We only need darwin + linux native builds.
- Locking `claude` to a specific version more strict than `>= 2.0.0` unless an integration test reveals a need.
