# Bun + TypeScript + Effect setup

## Objective

Stand up the TypeScript/Bun project structure. After this, `bun run src/main.ts ./WORKFLOW.md` starts a process that logs "symphony starting" and exits cleanly, and `bun test` runs (with zero tests) successfully.

## Context

### Parent PRD

- **PRD**: Symphony v1 (Effect.ts + Claude Code)
- **PRD Path**: `.claude/prds/symphony-v1/PRD.md`

### Related Tasks

- **Depends on**: Nix flake dev shell (so bun is on PATH).
- **Blocks**: every implementation task.

## Acceptance Criteria

- [ ] `package.json` declares dependencies: `effect`, `@effect/platform`, `@effect/platform-bun`, `@effect/schema`, `liquidjs`. Dev dependencies: `@effect/vitest`, `@types/bun`, `typescript`.
- [ ] `tsconfig.json` is strict: `"strict": true`, `"noUncheckedIndexedAccess": true`, `"exactOptionalPropertyTypes": true`, `"moduleResolution": "bundler"`, target `"ESNext"`, lib includes `"ESNext"`.
- [ ] `tsconfig.json` enforces the CLAUDE.md rules at the compiler level: `"noImplicitAny": true`. (No `any`, no `!` are linter concerns — see lint rule.)
- [ ] `bun.lockb` is checked in.
- [ ] `src/` skeleton matches PRD's Architecture file layout. Empty placeholder files are fine; each has a 2-line header comment per CLAUDE.md.
- [ ] `src/main.ts` is a minimal stub: parses argv for `<path-to-WORKFLOW.md>`, logs a startup line to stderr, exits 0. Wire later in the application-wiring task.
- [ ] `package.json` scripts: `dev` (`bun run --watch src/main.ts ./WORKFLOW.md`), `start` (`bun run src/main.ts`), `test` (`bun test`), `typecheck` (`tsc --noEmit`).

### Technical Constraints

- Use `import { … } from "effect"` (not `import * as Effect`) for namespace clarity.
- Pin Effect to a recent stable (>= 3.10 at time of writing). Verify `@effect/platform-bun` exists at the matching minor — if not, fall back to `@effect/platform` + `@effect/platform-node` (Bun has Node compat).
- ESM only. No `require`.
- Every TypeScript file starts with a 2-line header comment describing the file (CLAUDE.md rule).

### Relevant Code References

- PRD §Architecture → "Service graph (Effect Layers)" — informs the directory split.
- CLAUDE.md → "Critical Rules" — informs lint/tsconfig strictness.
